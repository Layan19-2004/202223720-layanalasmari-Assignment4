# AI Usage Report

## Overview

AI tools were used during the development of Assignment 4 to improve code quality, solve technical issues, enhance design decisions, and strengthen documentation.

The final project combined previous assignments into one polished portfolio web application. AI support was used as a learning and productivity tool while all final decisions, edits, testing, and submissions remained under my control.

All AI-generated suggestions were reviewed, modified, tested, and improved manually before use.

## Challenges and Limitations

AI was helpful, but not every suggestion was correct immediately.

### 1. Context Dependency

Some code suggestions were add-on blocks that depended on existing Assignment 3 code.

This required merging code correctly rather than replacing full files.

### 2. Generic Suggestions

Some ideas needed adjustment to better match the assignment requirements.

### 3. Need for Testing

Even good suggestions required manual browser testing before use.

### 4. Overengineering Risk

Some advanced ideas were unnecessary for a student portfolio project.

Simple and reliable solutions were preferred.

---

## How Problems Were Solved

To maintain quality, each suggestion was reviewed carefully.

Steps taken:

- Tested changes in browser
- Fixed file path issues
- Merged new code with old code properly
- Removed unnecessary complexity
- Adjusted design choices manually
- Compared work with assignment rubric

Only useful and working solutions were kept.

---

## Responsible Use and Academic Integrity

AI was used as a support tool, not as a replacement for learning.

I made sure to:

- Understand all code before using it
- Modify generated suggestions
- Test every feature manually
- Rewrite documentation in project-specific wording
- Keep control of final decisions
- Ensure originality of final submission

No AI output was copied blindly without review.
